# 独立 K 线看板

这是一个独立实现的网页版 K 线看板原型。目标是复刻主流加密货币交易所网页看板的功能结构和操作密度：行情标题栏、K 线图、均线、成交量、盘口、成交流、市场列表和只读下单模拟区。

本项目不复制币安的 logo、截图、图标、品牌文案、样式表或前端代码。界面采用独立命名、独立布局细节和独立实现；行情数据默认尝试读取公开市场接口，读取失败时自动切换到本地演示数据。

## 当前目标

- 尽量接近网页版 K 线看板的使用体验。
- 不使用任何交易所品牌资产。
- 不接入账户、不保存密钥、不提供真实下单能力。
- 只展示公开行情和本地计算结果。
- 保留清楚的版权、商标和风险边界。

## 如何运行

在本目录启动一个本地静态服务器：

```zsh
python3 -m http.server 4173
```

然后打开：

```text
http://127.0.0.1:4173
```

也可以直接打开 `index.html`，但本地服务器更接近真实浏览器环境，跨域和 WebSocket 行为更稳定。

## 已实现

- 多交易对切换：`BTCUSDT`、`ETHUSDT`、`BNBUSDT`、`SOLUSDT`、`XRPUSDT`。
- 多周期 K 线：`1m`、`5m`、`15m`、`1h`、`4h`、`1d`。
- Canvas K 线图：蜡烛图、影线、成交量、十字光标、价格轴、时间轴。
- 均线：`MA7`、`MA25`、`MA99`，可开关。
- 鼠标滚轮缩放图表密度。
- 盘口深度：买卖盘、价差、深度条。
- 实时成交流：方向、价格、数量、时间。
- 市场列表和 24h 指标。
- 只读下单模拟区：用于布局占位和数值计算，不会提交订单。
- 公开接口失败后的离线演示模式。

## 数据来源

程序会依次尝试这些公开市场数据域名：

- `https://data-api.binance.vision/api/v3`
- `https://api.binance.com/api/v3`

WebSocket 会尝试连接公开 market stream。网络不可用、地区限制、接口限速或 CORS 失败时，会进入本地演示数据模式。

1. 启动本地服务器。
2. 打开页面。
3. 切换交易对和周期。
4. 滚轮缩放图表。
5. 查看右侧盘口、成交流是否刷新。
6. 断网或接口失败时，确认页面仍显示演示行情，而不是白屏。
